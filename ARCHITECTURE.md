# Архитектура системы

## Обзор

Система оценки рыночной стоимости студентов объединяет данные из двух источников:

- **hh.ru** — вакансии с зарплатами и требуемыми навыками
- **Университет** — дисциплины, которые прошёл студент

Сопоставление происходит через **семантический поиск**: дисциплины и навыки
превращаются в векторы и сравниваются по смысловой близости.

## Потоки данных

### Парсинг вакансий

```
Пользователь → [запрос + кол-во] → FastAPI → hh.ru API
                                       ↓
                                  Вакансии + теги
                                   ↓           ↓
                              PostgreSQL     Qdrant
                           (вакансии, теги)  (эмбеддинги тегов)
```

### Оценка студента

```
Пользователь → [student_id] → FastAPI
                                  ↓
                            PostgreSQL: дисциплины студента
                                  ↓
                     ┌─── для каждой дисциплины ───┐
                     │                              │
                     │  Ollama: эмбеддинг           │
                     │     ↓                        │
                     │  Qdrant: top-K навыков       │
                     │     ↓                        │
                     │  PostgreSQL: ЗП по навыку    │
                     └──────────────────────────────┘
                                  ↓
                        Взвешенная оценка ЗП
                                  ↓
                            Ответ клиенту
```

## Сервисы (docker-compose)

| Сервис       | Образ                   | Порт  | Назначение |
|-------------|-------------------------|-------|------------|
| `app`       | Dockerfile (Python 3.12)| 8000  | FastAPI-приложение + фронтенд |
| `db`        | postgres:16-alpine      | 5432  | Хранение вакансий, тегов, студентов |
| `qdrant`    | qdrant/qdrant:v1.13.2   | 6333  | Векторная БД для семантического поиска |
| `ollama`    | ollama/ollama:latest    | 11434 | Генерация эмбеддингов (nomic-embed-text) |

---

## Стек технологий

| Слой          | Технология |
|---------------|------------|
| Backend       | FastAPI, SQLAlchemy (async), Pydantic |
| База данных   | PostgreSQL 16 + asyncpg |
| Векторная БД  | Qdrant (косинусное расстояние) |
| Эмбеддинги    | Ollama + nomic-embed-text (768-dim, мультиязычная) |
| Frontend      | Vanilla JS + Chart.js (SPA, тёмная тема) |
| Инфраструктура| Docker Compose, uv (менеджер пакетов) |

## API-эндпоинты

### Вакансии и теги

| Метод | Путь | Описание |
|-------|------|----------|
| POST  | `/api/v1/parse` | Парсинг вакансий с hh.ru |
| GET   | `/api/v1/vacancies` | Список вакансий с пагинацией |
| GET   | `/api/v1/tags` | Статистика тегов |

### Студенты

| Метод | Путь | Описание |
|-------|------|----------|
| GET   | `/api/v1/students/` | Список всех студентов |
| POST  | `/api/v1/students/` | Создание студента |
| GET   | `/api/v1/students/{id}` | Информация о студенте |
| POST  | `/api/v1/students/{id}/disciplines` | Добавление дисциплин |

### Оценка

| Метод | Путь | Описание |
|-------|------|----------|
| POST  | `/api/v1/students/{id}/evaluate?top_k=5` | Оценка стоимости студента |
| GET   | `/api/v1/students/{id}/skills?top_k=3` | Навыки студента в терминах hh.ru |
