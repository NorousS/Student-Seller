# PRD: HH.ru Student Evaluator (Система оценки рыночной стоимости студентов)

**Версия:** 2.0
**Статус:** Final
**Область:** EdTech / HR-Tech

---

## 1. Обзор продукта

### 1.1 Описание

**HH.ru Student Evaluator** — аналитическая платформа, которая оценивает потенциальную зарплату студента на основе его академической успеваемости и сопоставления дисциплин с реальными навыками, востребованными на рынке труда (данные с hh.ru).

### 1.2 Основная проблема

Студенты часто не понимают, как их оценки и предметы в вузе конвертируются в рыночную стоимость, а вузам сложно оценить, насколько их программы соответствуют требованиям ��аботодателей.

---

## 2. Целевая аудитория

- **Студенты:** Хотят знать свою «цену» на рынке и понимать, какие навыки приносят больше денег.
- **Методисты вузов:** Анализ соответствия курсов требованиям работодателей.
- **Рекрутеры:** Поиск талантливых выпускников с высоким «потенциальным чеком».

## 3. Функциональные требования

### 3.1 Сбор данных (Parsing)

- **Интеграция с hh.ru:** Автоматический сбор вакансий по заданным ключевым словам через `api.hh.ru/vacancies`.
- **Фильтрация по опыту:** Поддержка фильтра по уровню опыта (`noExperience`, `between1And3`, `between3And6`, `moreThan6`).
- **Обработка навыков:** Извлечение Key Skills из каждой вакансии, сохранение в PostgreSQL с дедупликацией, и параллельная индексация в Qdrant через эмбеддинги.
- **Зарплатная статистика:** Для расчётов используются только вакансии с зарплатой в рублях (`salary_currency = "RUR"`).

### 3.2 Математическая модель расчёта (Valuation Engine)

Оценка строится на четырёх этапах:

**Этап 1: Фильтрация по специальности**

Перед оценкой система семантически сопоставляет указанную специальность студента с поисковыми запросами в базе вакансий. Вакансии учитываются только если:

$$\text{similarity}(\text{specialty}, \text{search\_query}) \geq 0.7$$

Это гарантирует, что оценка основана на профильных вакансиях.

**Этап 2: Семантический поиск навыков**

Для каждой дисциплины студента:
1. Название дисциплины преобразуется в вектор через `nomic-embed-text` (768 измерений).
2. В Qdrant ищутся `top_k` (1–20, по умолчанию 5) ближайших навыков по косинусному расстоянию.
3. Отсеиваются навыки с `similarity < 0.6` (порог настраивается).
4. Отсеиваются навыки, встречающиеся менее чем в 3 вакансиях (`MIN_TAG_COUNT = 3`).
5. Удаляются вручную исключённые пользователем навыки.

**Этап 3: Вес навыка ($W_i$)**

Вес каждого навыка определяется тремя факторами:

$$W_i = \text{similarity}(d, s_i) \cdot \ln(1 + \text{count}_i) \cdot k_g$$

| Компонент | Описание |
|-----------|----------|
| $\text{similarity}(d, s_i)$ | Косинусное сходство между вектором дисциплины $d$ и навыка $s_i$ (0–1) |
| $\ln(1 + \text{count}_i)$ | Натуральный логарифм от числа вакансий с этим навыком. Сглаживает выбросы — навык из 1000 вакансий не получит вес в 10 раз больше навыка из 100 |
| $k_g$ | Коэффициент оценки: «5» = 1.0, «4» = 0.85, «3» = 0.75 |

**Этап 4: Итоговая оценка**

Единая взвешенная средняя по **всем навыкам всех дисциплин** студента:

$$\text{estimated\_salary} = \frac{\sum_{i} (\text{avg\_salary}_i \cdot W_i)}{\sum_{i} W_i}$$

где $\text{avg\_salary}_i$ — средняя зарплата по вакансиям, содержащим навык $i$.

**Уровень уверенности (Confidence):**

$$\text{confidence} = \frac{\text{matched\_disciplines}}{\text{total\_disciplines}}$$

- `matched_disciplines` — число дисциплин, для которых нашёлся хотя бы один навык выше всех порогов.
- Значение от 0.0 до 1.0.

### 3.3 Интерфейс (SPA, 4 вкладки)

#### Вкладка «📊 Оценка студента»
- Выбор студента из списка.
- Ввод специальности (текстовое поле).
- Фильтр по уровню опыта (5 уровней).
- Слайдер `top_k` (1–20).
- **Stat-карточки:** Оценочная зарплата (₽), Уверенность (%), Соотношение совпавших дисциплин.
- **Bar Chart:** Топ-12 навыков по вкладу в зарплату (Impact).
- **Radar Chart:** Сходство дисциплин и навыков.
- **Детальная таблица:** Все найденные навыки с similarity %, средней зарплатой, числом вакансий, кнопкой исключения (persistent state).
- **Бейджи оценок:** Цветовая кодировка (зелёный = 5, жёлтый = 4, красный = 3).
- Исключение/восстановление навыков вызывает мгновенный пересчёт.

#### Вкладка «👩‍🎓 Студенты»
- Создание студента (ФИО, группа).
- Добавление дисциплин с выбором оценки (3/4/5, по умолчанию 5).
- Добавление дисциплин к существующим студентам (идемпотентно).
- Список студентов с бейджами дисциплин.

#### Вкладка «🔍 Парсинг вакансий»
- Поисковый запрос (по умолчанию: «python»).
- Выбор количества (1–100, по умолчанию 20).
- Фильтр по опыту.
- **Stat-карточки:** Спарсено вакансий, средняя зарплата.
- **Bar Chart:** Топ навыков из текущего парсинга (без навыков с count = 1).
- **Лог парсинга:** Детальный список вакансий с зарплатами.

#### Вкладка «🏷️ Навыки и теги»
- Поиск по названию вакансии.
- **Stat-карточки:** Всего вакансий, всего тегов, средняя зарплата.
- **Pie Chart:** Топ-20 навыков по распределению.
- **Таблица тегов:** Все теги с количеством вакансий и процентами, прогресс-бар.

#### Общие особенности UI
- Тёмная тема (GitHub-стиль).
- Адаптивная сетка (2 колонки на десктопе, 1 на мобильном).
- Все графики — Chart.js (Bar, Radar, Pie).

## 4. API-эндпоинты

### Вакансии и теги

| Метод | Путь | Параметры | Описание |
|-------|------|-----------|----------|
| POST | `/api/v1/parse` | `query` (str), `count` (1–100, default 50), `experience` (enum, optional) | Парсинг вакансий с hh.ru |
| GET | `/api/v1/vacancies` | `query` (str, optional), `experience` (enum, optional), `limit` (default 100) | Список вакансий с пагинацией |
| GET | `/api/v1/tags` | — | Статистика всех тегов |

### Студенты

| Метод | Путь | Параметры | Описание |
|-------|------|-----------|----------|
| GET | `/api/v1/students/` | — | Список всех студентов |

| POST | `/api/v1/students/{id}/disciplines` | `disciplines[]` (name, grade) | Добавление/обновление дисциплин (идемпотентно) |

### Qdrant (коллекция `hh_skills`)

```
Point {
  id: int64           — детерминированный хэш от названия: abs(hash(name)) % 2^63
  vector: float[768]  — эмбеддинг от nomic-embed-text
  payload: {
    name: string      — название навыка ("Python", "Docker", ...)
  }
}
```

## 6. Техническая спецификация

| Компонент | Технология | Примечание |
|-----------|------------|------------|
| **Backend** | FastAPI | Асинхронный ASGI-сервер (uvicorn) |
| **ORM** | SQLAlchemy 2.0 (async) + asyncpg | Асинхронная работа с PostgreSQL |
| **Validation** | Pydantic + Pydantic Settings | Валидация данных и конфигурация |
| **HTTP Client** | httpx | Асинхронные запросы к hh.ru API |
| **Primary DB** | PostgreSQL 16 (alpine) | Хранение вакансий, тегов, студентов |
| **Vector DB** | Qdrant v1.13.2 | Хранение и поиск эмбеддингов навыков (косинусное расстояние) |
| **LLM / Embeddings** | Ollama + nomic-embed-text | Локальная генерация 768-мерных эмбеддингов (мультиязычная модель) |
| **Frontend** | Vanilla JS + Chart.js | Лёгкое SPA, тёмная тема, 4 вкладки |
| **Package Manager** | uv | Быстрая установка Python-зависимостей |
| **Deployment** | Docker Compose | 4 сервиса: app (:8000), db (:5432), qdrant (:6333), ollama (:11434) |

## 9. Риски и ограничения

### Риски

| Риск | Митигация |
|------|-----------|
| Низкое качество данных (отсутствие зарплат в вакансиях) | Учитываются только вакансии с указанной зарплатой в рублях |
| Семантический шум (неправильное сопоставление предметов) | Ручная фильтрация навыков пользователем + пороги similarity |
| Таймаут Ollama при первой загрузке модели | Ollama автоматически скачивает модель при первом запросе (таймаут 300 сек) |

### Текущие ограничения

- Нет аутентификации и авторизации (публичный API).
- Нет пагинации для списка студентов.
- Нет эндпоинта удаления студента.
- Нет кэширования эмбеддингов (пересчёт при каждом запросе оценки).
- Состояние фронтенда не сохраняется между перезагрузками страницы.
- Нет rate-limiting на эндпоинте парсинга.
