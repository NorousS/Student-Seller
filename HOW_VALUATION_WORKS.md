# Как рассчитывается стоимость студента

## Общая идея

Система оценивает **потенциальную рыночную стоимость** студента, сопоставляя его
академические дисциплины с навыками, которые требуются в реальных вакансиях на hh.ru.

Для каждой дисциплины находится набор наиболее похожих навыков (тегов) из вакансий,
а затем на основе зарплат в этих вакансиях вычисляется взвешенная оценка.

---

## Пошаговый алгоритм

### 1. Получение дисциплин студента

Из базы данных загружается список дисциплин (`Discipline`), привязанных к студенту
через таблицу `StudentDiscipline`. Например:

```
["Программирование на Python", "Базы данных", "Машинное обучение"]
```

### 2. Семантический поиск навыков hh.ru

Для **каждой дисциплины** выполняется семантический поиск в векторной базе Qdrant:

1. Название дисциплины сначала нормализуется в `app/embeddings.py` (`trim + collapse spaces + lowercase`), затем преобразуется в вектор (эмбеддинг) через модель `nomic-embed-text`
2. В коллекции `hh_skills` ищутся `top_k` ближайших векторов (по косинусному расстоянию)
3. Отсеиваются результаты с `similarity < similarity_threshold` (по умолчанию 0.6)

Результат — список пар `(навык_hh, similarity_score)`:

```
"Программирование на Python" → [
  ("Python", 0.92),
  ("Django", 0.78),
  ("FastAPI", 0.75),
  ("Flask", 0.71),
  ("Backend", 0.65)
]
```

### 3. Получение зарплатной статистики

Для каждого найденного навыка из PostgreSQL извлекается:

- **Средняя зарплата** (`avg_salary`) — среднее значение `salary_from` (или `salary_to`,
  если `salary_from` не указана) по всем вакансиям с этим тегом. Учитываются
  **только вакансии с зарплатой в рублях** (`salary_currency = "RUR"`).
- **Количество вакансий** (`vacancy_count`) — сколько вакансий содержат этот тег
  и имеют указанную зарплату.
- Дополнительно применяются фильтры оценки: по `search_query` (семантически
  релевантные специальности) и `experience` (если передан в запросе).

### 4. Расчёт взвешенной оценки

Формула итоговой оценки:

```
                    Σ (avg_salary_i × weight_i)
estimated_salary = ─────────────────────────────
                         Σ (weight_i)
```

где вес каждого навыка вычисляется по выбранной формуле (см. "Формулы расчёта весов" ниже).

В расчёт входят только **валидные** совпадения: `avg_salary != null`,
`vacancy_count >= 3` и `excluded = false`.

#### Формулы расчёта весов

Система поддерживает 6 различных формул для расчёта веса навыка. Каждая формула
по-разному балансирует между схожестью, частотностью и оценкой:

| Формула | Формула веса | Особенности |
|---------|--------------|-------------|
| **baseline** | `similarity × log(1 + vacancy_count) × grade_coeff` | Базовая формула (по умолчанию). Логарифмически учитывает частотность навыка |
| **linear** | `similarity × (vacancy_count / max_count) × grade_coeff` | Линейная зависимость от частотности, нормализованная к максимуму |
| **quadratic** | `similarity² × log(1 + vacancy_count) × grade_coeff` | Усиливает высокие совпадения (similarity²), снижает вес средних |
| **exponential** | `exp(similarity - 0.5) × log(1 + vacancy_count) × grade_coeff` | Экспоненциально усиливает совпадения выше 0.5 |
| **tfidf** | `similarity × log(total_vacancies / vacancy_count) × grade_coeff` | TF-IDF подход: предпочитает редкие узкоспециализированные навыки |
| **matrix** | `similarity × correlation_boost × log(1 + vacancy_count) × grade_coeff` | Учитывает корреляции между навыками (повышает вес сопутствующих навыков) |

#### Выбор формулы

Формулу можно указать через параметр `formula` в API:
- `POST /api/v1/profile/student/evaluate?formula=quadratic`
- `POST /api/v1/students/{id}/evaluate?formula=tfidf`

Список доступных формул: `GET /api/v1/profile/student/formulas`

Для сравнения всех формул на одном студенте (только admin):
- `POST /api/v1/analysis/compare/{student_id}?specialty=Python developer`

#### Почему разные формулы?

| Компонент         | Смысл |
|-------------------|-------|
| `similarity`      | Чем точнее дисциплина соответствует навыку, тем больше его вклад |
| `log(1 + count)`  | Частотность навыка: чем чаще он встречается в вакансиях, тем надёжнее статистика по зарплате. Логарифм сглаживает выбросы — навык, встретившийся 1000 раз, не в 1000 раз важнее навыка, встретившегося 10 раз |
| `grade_coeff`     | Учитывает академическую успеваемость: `3 → 0.75`, `4 → 0.85`, `5 → 1.0` |

### 5. Расчёт уверенности

```
score_i = similarity_i × grade_coeff_i
confidence = average(top-3 score_i среди валидных совпадений)
```

- В confidence попадают только валидные совпадения (`avg_salary != null`, `vacancy_count >= 3`, `excluded = false`)
- Используется фиксированный top-3 лучших совпадений (или меньше, если валидных совпадений < 3)
- Если валидных совпадений нет, `confidence = 0.0`

## Пользовательское управление

### Выбор количества навыков (top_k)

Пользователь может управлять параметром `top_k` через слайдер в UI (от 1 до 20).
Этот параметр определяет, сколько ближайших навыков hh.ru искать на каждую дисциплину.

- **Малое `top_k`** (1–2) — только самые релевантные навыки, высокая точность, меньше шума
- **Большое `top_k`** (10–20) — более полный охват, но могут попасть менее релевантные навыки

### Исключение навыков (excluded_skills)

Пользователь может исключить конкретные навыки из расчёта, нажав кнопку 🗑️ в таблице
детальной разбивки.

Когда навык исключён:
1. Он помечается `excluded: true` и остаётся в ответе для отображения
2. Его вес (`weight_i`) **не учитывается** в формуле взвешенной оценки
3. Стоимость пересчитывается автоматически
4. В таблице строка показывается затемнённой с кнопкой восстановления ♻️

Это полезно, когда семантический поиск нашёл нерелевантный навык
(например, «Коммуникации» для дисциплины «Компьютерные сети»).

## Пример

Студент с дисциплинами: **Python**, **SQL**, **Философия**.

| Дисциплина | Навык hh.ru | Similarity | Avg Salary | Vacancies | Weight |
|------------|-------------|------------|------------|-----------|--------|
| Python     | Python      | 0.95       | 180 000    | 500       | 5.90   |
| Python     | Django      | 0.80       | 160 000    | 200       | 4.24   |
| SQL        | SQL         | 0.93       | 150 000    | 400       | 5.57   |
| SQL        | PostgreSQL  | 0.82       | 170 000    | 300       | 4.68   |
| Философия  | —           | —          | —          | —         | —      |

```
estimated_salary = (180000×5.90 + 160000×4.24 + 150000×5.57 + 170000×4.68)
                 / (5.90 + 4.24 + 5.57 + 4.68)
               ≈ 165 800 ₽

confidence = average(top-3(similarity × grade_coeff)) = (0.95 + 0.93 + 0.82) / 3 = 0.90
```

## Конфигурация

| Параметр               | Значение по умолчанию | Описание |
|------------------------|-----------------------|----------|
| `similarity_threshold` | 0.6                   | Минимальный порог косинусного сходства |
| `top_k`                | 5 (параметр запроса, 1–20)  | Сколько навыков искать на каждую дисциплину |
| `excluded_skills`      | `[]` (параметр запроса) | Навыки для исключения из расчёта |
| `embedding_model`      | `nomic-embed-text`    | Модель для генерации эмбеддингов |
| `embedding_dimension`  | 768                   | Размерность вектора |
| `formula`              | `baseline` (параметр запроса) | Формула расчёта веса (baseline, linear, quadratic, exponential, tfidf, matrix) |
| `grade_3_coeff` | 0.75 | Коэффициент для оценки 3 |
| `grade_4_coeff` | 0.85 | Коэффициент для оценки 4 |
| `grade_5_coeff` | 1.0  | Коэффициент для оценки 5 |

## Расширенная оценка: факторная декомпозиция

### Новый эндпоинт

`POST /api/v1/evaluation/{student_id}/evaluate-enhanced`

В дополнение к стандартной оценке (`estimated_salary`, `confidence`), расширенная оценка возвращает **факторную декомпозицию** — процентный вклад каждого фактора в итоговую стоимость.

### Три фактора

Каждый навык в оценке получает вес по формуле:

```
weight = similarity × log(1 + vacancy_count) × grade_coeff
```

Расширенная оценка разбивает этот вес на **три составляющие** и вычисляет их процентный вклад:

| Фактор | Формула | Описание |
|--------|---------|----------|
| `similarity_contribution` | `similarity / (similarity + log1p + grade)` | Насколько точно дисциплина соответствует навыку |
| `vacancy_demand_contribution` | `log1p(vacancy_count) / (similarity + log1p + grade)` | Насколько востребован навык на рынке |
| `academic_grade_contribution` | `grade_coeff / (similarity + log1p + grade)` | Вклад академической оценки студента |

### Модель ответа

```json
{
  "estimated_salary": 165800,
  "confidence": 0.90,
  "factor_breakdown": {
    "similarity_pct": 42.3,
    "vacancy_demand_pct": 38.1,
    "academic_grade_pct": 19.6
  },
  "matches": [...]
}
```

### Интерпретация

- **Высокий similarity_pct** — оценка определяется прежде всего точностью совпадения дисциплин с рыночными навыками
- **Высокий vacancy_demand_pct** — доминирует востребованность навыков (много вакансий с высокими зарплатами)
- **Высокий academic_grade_pct** — академические оценки существенно влияют на результат

### Экспорт результатов

`GET /api/v1/evaluation/{student_id}/export` — возвращает полный JSON с оценкой, декомпозицией и метаданными.

## Компетенции и категоризация

### Категории дисциплин

Каждая дисциплина может иметь категорию компетенции (поле `Discipline.category`):
- `programming` — программирование
- `databases` — базы данных
- `math` — математика
- `management` — менеджмент
- `networks` — сети
- `security` — безопасность
- `ai_ml` — искусственный интеллект
- и другие

### Автоматическая категоризация

Сервис `app/categorization.py` использует эмбеддинги Ollama для автоматического определения категории:
1. Генерирует вектор названия дисциплины (nomic-embed-text)
2. Сравнивает с эталонными векторами категорий (cosine similarity)
3. Присваивает категорию с наибольшим сходством

### Блоки компетенций

Сервис `app/competence.py` группирует дисциплины по категориям для визуализации:
- Агрегация `aggregate_by_competence(disciplines)` → список `CompetenceBlock`
- Каждый блок: название категории, список дисциплин, средний similarity

### Влияние на оценку

Категоризация не влияет на числовую оценку напрямую — она используется для:
- Визуальной группировки навыков в UI
- Административного анализа программы обучения
- Ручного переопределения через `PATCH /api/v1/admin/partnership/disciplines/{id}/category`

## Ключевые файлы

- `app/valuation.py` — основной алгоритм оценки (`evaluate_student`)
- `app/formulas/` — реализации всех 6 формул расчёта весов
- `app/formulas/__init__.py` — FormulaRegistry для выбора формулы
- `app/analysis/` — метрики качества формул и генератор тестовых данных
- `app/grouping/` — кластеризация навыков и дисциплин по категориям
- `app/vector_store.py` — семантический поиск в Qdrant (`search_similar_skills`)
- `app/embeddings.py` — генерация эмбеддингов через Ollama
- `app/routers/evaluation.py` — API-эндпоинты `/{student_id}/evaluate` и `/{student_id}/skills`
- `app/routers/analysis.py` — API для сравнения формул и кластеризации (admin)
- `app/categorization.py` — LLM-категоризация дисциплин через Ollama
- `app/competence.py` — агрегация блоков компетенций
- `app/routers/admin_disciplines.py` — API категоризации (admin)
