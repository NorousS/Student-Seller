# Как работают теги (навыки)

## Откуда берутся теги

Теги — это **ключевые навыки**, которые hh.ru прикрепляет к каждой вакансии.
Они извлекаются автоматически при парсинге вакансий через API hh.ru.

### Процесс сбора тегов

```
Запрос на парсинг  →  API hh.ru  →  Список вакансий  →  Теги из каждой вакансии
       ↓                                                         ↓
   PostgreSQL  ←──────────────────────────────────── Таблица Tag + связи
       ↓
   Qdrant  ←──────── Эмбеддинги тегов (для семантического поиска)
```

1. Пользователь вводит поисковый запрос (например, «Python разработчик») и количество
   вакансий в окне **Парсинг** на фронтенде
2. Сервер отправляет запрос к `api.hh.ru/vacancies` с указанными параметрами
3. Для каждой вакансии извлекаются теги — поле `key_skills` в ответе API
4. Каждый тег сохраняется в таблицу `Tag` (с дедупликацией по имени)
5. Создаётся связь вакансия ↔ тег в таблице `vacancy_tag_association`
6. **Параллельно** все уникальные теги индексируются в векторной БД Qdrant

### Пример тегов из одной вакансии

```json
{
  "name": "Python-разработчик",
  "salary_from": 150000,
  "key_skills": ["Python", "Django", "PostgreSQL", "Docker", "Git", "REST API"]
}
```

Каждый элемент из `key_skills` становится отдельным тегом.

## Индексация в Qdrant

При парсинге каждый **новый** тег проходит следующий путь:

1. Проверяется, есть ли уже такой тег в коллекции `hh_skills` (по хэшу имени)
2. Если нет — название тега отправляется в **Ollama** (`nomic-embed-text`)
3. Модель возвращает **вектор из 768 чисел** — числовое представление смысла навыка
4. Вектор сохраняется в Qdrant с `payload = {"name": "Python"}`

```
"Python"     → [0.12, -0.34, 0.56, ..., 0.78]  (768 чисел)
"Django"     → [0.15, -0.28, 0.61, ..., 0.42]
"PostgreSQL" → [0.08, -0.41, 0.33, ..., 0.91]
```

Идентификатор точки в Qdrant — **детерминированный хэш** от названия:
```python
point_id = abs(hash("Python")) % (2**63)
```
Это обеспечивает дедупликацию: повторная индексация того же навыка не создаёт дубликатов.

## Семантический поиск

Когда нужно найти навыки hh.ru, похожие на дисциплину студента:

1. Название дисциплины (например, «Программирование на Python») → эмбеддинг через Ollama
2. Qdrant ищет ближайшие векторы по **косинусному расстоянию** (`Distance.COSINE`)
3. Возвращаются `top_k` результатов с оценкой сходства (`score` от 0 до 1)

```
Запрос: "Программирование на Python"

Результаты:
  Python      → 0.92  ✓ (выше порога 0.6)
  Django      → 0.78  ✓
  FastAPI     → 0.75  ✓
  Flask       → 0.71  ✓
  JavaScript  → 0.45  ✗ (ниже порога — отсеивается)
```

### Почему семантический, а не точный поиск?

Дисциплины в университете называются иначе, чем навыки на hh.ru:

| Дисциплина              | Навыки hh.ru                 |
|-------------------------|------------------------------|
| Базы данных             | SQL, PostgreSQL, MySQL       |
| Объектно-ориентированное программирование | Java, C++, ООП |
| Веб-разработка          | HTML, CSS, JavaScript, React |
| Математический анализ   | Аналитика, Data Science      |

Семантический поиск понимает **смысл**, а не совпадение букв.

## Отображение тегов на фронтенде

### Вкладка «🏷️ Теги»

Отображает статистику **всех** тегов из базы данных (из всех парсингов).

#### Doughnut-диаграмма

- Показывает **только значимые теги** — те, что встречаются в **более чем 1 вакансии**
- Каждый тег отображается отдельным сегментом (без группировки в «Другие»)
- При наведении — тултип с количеством вакансий
- Если тегов больше 15 — легенда скрывается (остаётся только тултип)

#### Таблица тегов

- Показывает **все** теги, отсортированные по количеству вакансий
- Теги с `count ≤ 1` визуально приглушены (`opacity: 0.45`)
- Для каждого тега — прогресс-бар относительно самого популярного

### Вкладка «📊 Парсинг»

При запуске парсинга показывается:

- **Bar-chart** — топ тегов из **текущего** парсинга (без тегов с `count = 1`)
- **Лог** — детальный список спарсенных вакансий с их зарплатами

## Структура данных

### PostgreSQL

```
Tag                          vacancy_tag_association         Vacancy
┌────────────┐               ┌──────────────────┐           ┌───────────────┐
│ id (PK)    │──────────────│ tag_id (FK)       │──────────│ id (PK)       │
│ name       │               │ vacancy_id (FK)   │           │ title         │
└────────────┘               └──────────────────┘           │ salary_from   │
                                                            │ salary_to     │
                                                            │ salary_currency│
                                                            └───────────────┘
```

### Qdrant (коллекция `hh_skills`)

```
Point {
  id: int64           — хэш от названия навыка
  vector: float[768]  — эмбеддинг от nomic-embed-text
  payload: {
    name: string      — название навыка ("Python", "Docker", ...)
  }
}
```

## Ключевые файлы

- `app/routers/vacancies.py` — парсинг вакансий, извлечение и сохранение тегов
- `app/vector_store.py` — индексация и поиск в Qdrant
- `app/embeddings.py` — генерация эмбеддингов через Ollama
- `app/models.py` — SQLAlchemy-модели `Tag`, `Vacancy`, связующая таблица
- `app/static/index.html` — фронтенд (вкладки Теги и Парсинг)
