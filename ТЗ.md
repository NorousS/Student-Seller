# Техническое задание: Доработка платформы HH.ru Student Evaluator

## 1. Целевые обновления функционала

### 1.1. Публичная витрина (Landing Page) и SEO
*   **Концепция:** Посадочная страница должна быть оптимизирована для поисковых систем и сфокусирована на быстрой «продаже» выпускников работодателям, у которых «горит» производство.
*   **Первый экран:** Блок с 5 анонимными профилями лучших студентов.
    *   **Состав карточки:** Анонимное фото, потенциальная цена (зарплата), краткий профиль (не просто специальность, а компетенции, например: «Инженер-радиоэлектрон с пониманием систем навигации»).
    *   **Призыв к действию:** Кнопка «Пригласить на собеседование».

### 1.2. Система доступа и монетизации (Партнерство с вузом)
Система должна разделять работодателей на две категории:
1.  **Партнеры вуза (КМПО, КВЗ и др.):**
    *   Кнопка «Пригласить на собеседование» активна.
    *   Полный доступ к анонимизированному профилю (достижения, курсовые, блоки знаний).
2.  **Сторонние организации:**
    *   При попытке пригласить студента выводится модальное окно с предложением «заплатить деньги» или «заключить договор с вузом».
    *   Доступ к прямым контактам студента (телефон, email) закрыт для всех до этапа принятия запроса студентом.

### 1.3. Интерфейс профиля студента «на языке работодателя»
*   **Группировка знаний:** Вместо перечисления отдельных вузовских дисциплин система должна объединять их в **укрупненные блоки компетенций** (например, блок «САПР», включающий три дисциплины со средним баллом 4.8).
*   **Визуализация качества:**
    *   Работодатель **не должен видеть** полную академическую ведомость (все оценки).
    *   Акцент на сильных сторонах: «4 дисциплины сданы на отлично», «защищены 2 курсовые по тематике X», победы в стартапах и софтскилы.
    *   Отображение даты готовности к работе (например, «6 июля получит диплом и готов выйти»).

Отлично, давай детализируем алгоритм объединения академических дисциплин в **укрупненные блоки компетенций**. Это позволит работодателю видеть не разрозненный список предметов, а готовую «картину» навыков студента на понятном рынке языке.

### Алгоритм формирования блоков компетенций

Для реализации этой функции в проект необходимо добавить слой семантической группировки.

#### 1. Классификация и группировка
Вместо ручного сопоставления каждой дисциплины, система будет использовать **LLM (Ollama)** для категоризации:
*   **Процесс:** Для каждой дисциплины студента (например, «Алгоритмы», «Язык Python», «Базы данных») агент генерирует общую категорию или «Компетенцию» (например, «Software Development»).
*   **Хранение:** В БД добавляется сущность `CompetenceBlock`, которая объединяет группу `Disciplines`.

#### 2. Расчет агрегированных метрик блока
Для каждого блока компетенций рассчитываются следующие показатели:
*   **Средний балл блока ($G_{avg}$):** Среднее арифметическое оценок всех дисциплин, входящих в блок.
    $$G_{avg} = \frac{\sum \text{grade}_i}{n}$$
*   **Рыночная ценность блока:** Сумма оценочных стоимостей всех дисциплин блока, рассчитанная по основной формуле.
*   **Сильные стороны (Strong Points):** Автоматический подсчет количества дисциплин в блоке, сданных на «отлично», и связанных с ними курсовых работ.

#### 3. Формирование «карточки компетенции» для работодателя
Работодатель видит в профиле студента не ведомость, а набор карточек:
1.  **Название блока:** Например, «Системы автоматизированного проектирования (САПР)».
2.  **Уровень владения:** Визуализируется через средний балл (например, 4.8) или Radar Chart сходства с рынком.
3.  **Ключевые навыки:** Топ-3 тега с hh.ru, которые имеют наибольший вес ($W_i$) внутри этого блока дисциплин.
4.  **Достижения:** Текстовая выжимка (например, «Включает 3 профильные дисциплины, 2 курсовых проекта»).

### Изменения в логике отображения (Role-Based Visibility)
Система должна применять разные правила отображения данных в зависимости от роли:
*   **Студент/Админ:** Видят полный список дисциплин, оценки и возможность исключения навыков.
*   **Работодатель (анонимно):** Видит только названия укрупненных блоков, их среднюю оценку и потенциальную стоимость.
*   **Работодатель (партнер):** Получает доступ к детальному описанию компетенций и возможности пригласить на собеседование.
